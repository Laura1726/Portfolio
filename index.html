Bienvenue sur mon portfolio de BUT Informatique

Qui suis-je ?

Je m'appelle Laura Bornert, étudiante en BUT Informatique à l'IUT de Belfort, actuellement en fin de première année.

Ce site web a pour objectif de présenter une sélection de projets réalisés durant l'année universitaire 2024-2025, et surtout de montrer les compétences que j'ai développées au fil de ces expériences.

Pourquoi ce portfolio ?

L'objectif principal de ce portfolio n'est pas seulement de montrer ce que j'ai produit, mais de prouver que je maîtrise certains savoirs et savoir-faire essentiels en informatique, à travers l'analyse de projets concrets.

Pour cela, j'ai sélectionné deux compétences du programme de BUT Informatique, que j'ai mises en œuvre dans le cadre de deux SAÉ (Situations d'Apprentissage et d'Évaluation) réalisées en équipe.

Compétence UE4 - Gérer des données de l'information

SAÉ 2.04 - Création d'un site de e-commerce de vente de meubles (Naturalis)

Figure 1 : Création des tables SQL pour le site Naturalis

La Figure 1 montre une partie du fichier sql_projet.sql, que j'ai moi-même rédigé dans le cadre du projet Naturalis, une application web de vente de meubles. L'objectif était de créer une base de données, c'est-à-dire un "espace organisé" dans lequel on stock toutes les informations nécessaires au bon fonctionnement du site : les utilisateurs, les produits, les commandes, etc.

Avant de pouvoir utiliser une base de données, il faut la concevoir : cela consiste à décider quelles tables on aura (par exemple une table utilisateur pour les clients et administrateurs), quels types de données on va stocker (texte, nombre, date, etc.), et comment lier ces tables entre elles.

Avant de pouvoir l’utiliser, il faut concevoir cette base : cela signifie décider quelles données doivent être enregistrées , dans quelles tables, sous quelle forme (texte, nombre, booléen…), et comment les différentes tables sont reliées entre elles. Dans l’exemple de la figure, on voit notamment la table utilisateur avec des colonnes comme login , email , ou role .

Savoirs :

  • Compréhension du rôle d’une base de données relationnelle dans une application web.
  • Connaissance des types de données SQL (VARCHAR, INT, BOOLEAN…).
  • Notion de clé primaire et de contraintes d’intégrité (ex. NOT NULL, UNIQUE).
  • Traduction d'un besoin fonctionnel en structure de base de données

Savoir-faire :

  • Écriture d’instructions SQL pour créer les tables et supprimer les anciennes (DROP TABLE, CREATE TABLE).
  • Utilisation de contraintes et types de champs pour garantir la cohérence des données
  • Préparation d'un environnement de base de données propre et réutilisable grâce à la clause IF NOT EXISTS
  • Organisation logique de la base pour qu’elle soit facilement exploitable par les développeurs.

Figure 2 : Exécution d'une requête SQL depuis une application Flask (Python)

La Figure 2 montre le lien entre la base de données et le code Python du site Naturalis.

Ici, il s’agit d’un contrôleur , c’est-à-dire un composant du site web qui va chercher les données dans la base et les afficher à l’utilisateur connecté.

La fonction affichée (client_coordonnee_show) permet par exemple de récupérer les informations d’un utilisateur à partir de son identifiant de session, puis d’exécuter une requête SQL pour afficher les adresses et le nombre de commandes liées à chaque adresse.

Savoirs :

  • Compréhension du fonctionnement d’un contrôleur dans un site web dynamique.
  • Notion de sécurité des requêtes SQL (prévention des injections avec les requêtes paramétrées).
  • Compréhension de la jointure logique entre les tables (ici : utilisateur, adresse, commande).
  • Notions de calculs en base de données avec des fonctions comme COUNT() ou DISTINCT.

Savoir-faire :

  • Connexion à une base de données depuis du code Python.
  • Écriture de requêtes SQL avancées et intégration dans le code via des paramètres (%s).
  • Exploitation des résultats SQL dans le programme (fetchone()).
  • Adaptation de la logique métier au contexte utilisateur (affichage dynamique selon session).

Cette figure illustre ma capacité à extraire et manipuler des données en fonction de l'utilisateur connecté, ce qui est fondamental pour tout site dynamique.

Compétence UE6 - Travailler dans une équipe informatique

SAÉ 1.05-1.06 - Créer un site web d'une entreprise d'informatique (Thalès entreprise)

Figure 1 : Visualisation des contributions GitHub de l'équipe

La Figure 1 montre des statistiques GitHub et mes contributions sur la période de septembre 2024 à mai 2025. On y observe que j'ai effectué 34 commits, avec 638 lignes ajoutées et 205 supprimées. Ces chiffres montrent que j'ai été un membre actif, en particulier durant la phase de développement en octobre 2024.

Savoirs :

  • Principe de versioning collaboratif
  • Utilité des commits et du suivi de l'historique des modifications
  • Règles de nommage, documentation dans les commits

Savoir-faire :

  • Utiliser GitHub pour versionner un projet en équipe
  • Partager, synchroniser et intégrer du code (push/pull/merge)
  • Analyser les statistiques de contribution pour répartir équitablement les tâches

Figure 2 : Utilisation de Discord pour la gestion collaborative

La Figure 2 montre notre serveur Discord dédié à la SAÉ S1.05-S1.06. L'organisation en canaux thématiques (#discussion, #répartition-pages, #zoning-wireframe-style-tile, #commandes-git) nous a permis de structurer la communication et d'éviter que les messages importants soient noyés dans des discussions générales.

Savoirs :

  • Organisation de l'information par canaux thématiques
  • Utilisation d'outils collaboratifs en ligne
  • Importance de la communication asynchrone dans un projet informatique

Savoir-faire :

  • Utiliser Discord pour organiser et suivre le travail d'équipe
  • Partager des ressources, poser des questions techniques, coordonner les tâches
  • Structurer un espace de discussion pour éviter la confusion et centraliser l'information
